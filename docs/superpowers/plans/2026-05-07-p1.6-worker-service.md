# P1.6 — Worker as a Service Implementation Plan

> Polish/integration to P1.5 — no new design. Brainstorming skipped.

**Goal:** 把 `radar scheduler serve` 跑成 docker-compose 服务（自动重启、与基础设施同生命周期）；GitHub API 提速到 5000/hr；Langfuse trace 端到端验证可见。

**Tech Stack:** Docker / docker-compose；已有 Python 栈不变。

## Task 3: docker-compose radar-worker service

**Files:**
- Modify: `F:/codes/ISBE/docker-compose.yml`

- [ ] Add `radar-worker` service before the `volumes:` section:

```yaml
  radar-worker:
    build:
      context: .
      dockerfile: Dockerfile
    image: isbe/radar:latest
    container_name: isbe-radar-worker
    depends_on:
      postgres:
        condition: service_healthy
      prefect-server:
        condition: service_started
      minio:
        condition: service_healthy
    environment:
      # DB / infra (override host defaults to use compose service names)
      POSTGRES_HOST: postgres
      POSTGRES_PORT: "5432"
      POSTGRES_USER: ${POSTGRES_USER:-isbe}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-changeme}
      POSTGRES_DB: ${POSTGRES_DB:-isbe}
      MINIO_ENDPOINT: minio:9000
      MINIO_ROOT_USER: ${MINIO_ROOT_USER:-isbe}
      MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD:-changeme123}
      PREFECT_API_URL: http://prefect-server:4200/api
      LANGFUSE_HOST: http://langfuse:3000
      LANGFUSE_PUBLIC_KEY: ${LANGFUSE_PUBLIC_KEY:-}
      LANGFUSE_SECRET_KEY: ${LANGFUSE_SECRET_KEY:-}
      # LLM
      ISBE_LLM_PROVIDER: ${ISBE_LLM_PROVIDER:-deepseek}
      ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY:-}
      DEEPSEEK_API_KEY: ${DEEPSEEK_API_KEY:-}
      # GitHub auth
      GITHUB_TOKEN: ${GITHUB_TOKEN:-}
      # ISBE
      ISBE_UID: ${ISBE_UID:-me}
    volumes:
      # Bind mount user-visible state (paths inside container = paths on host)
      - ./memory:/app/memory
      - ./papers:/app/papers
      - ./artifacts:/app/artifacts
      # Bind mount source for live edits (no rebuild needed for code changes)
      - ./src:/app/src
      - ./alembic:/app/alembic
      - ./alembic.ini:/app/alembic.ini
    command: ["scheduler", "serve"]
    restart: unless-stopped
```

> **Note**: Bind mount `./src` over the image's `/app/src` means container always uses host source. For production builds (frozen image), remove the `./src` bind mount — but for dev, it lets `git pull` reflect immediately.

## Self-Review

1. **Spec coverage**: P1.5 user complaint "scheduler 必须自跑长进程" is fixed by docker-compose service. GITHUB_TOKEN is a polish that unblocks T4 from P1.5. Langfuse verification closes the trace loop — but **only if user creates project** in UI; we can't auto-create via API in this scope.
2. **No design choices**: all tasks are mechanical.
3. **Risk**: Dockerfile build ~2min first time (uv sync inside slim Python). User must wait once. Subsequent rebuilds cached.
4. **Risk 2**: Bind-mounting `./src` means host code changes affect the worker without rebuild — convenient for dev, but means "production-frozen" requires removing that mount. Document as a comment in compose.
5. **Out of scope**: GitHub OAuth flow (we use Personal Access Token), multi-user, secrets manager.
